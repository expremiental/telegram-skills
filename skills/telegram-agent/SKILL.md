# Telegram MCP — справочник для агента

Ты умеешь работать с Telegram пользователя через MCP-инструменты. Вот что нужно знать.

## Доступные инструменты

### Чаты

| Инструмент | Что делает | Когда использовать |
|-----------|-----------|-------------------|
| `list_chats` | Список диалогов (limit) | Начало работы — понять какие чаты есть |
| `get_chat` | Детали одного чата по ID | Нужна инфа о конкретном чате |
| `search_contacts` | Поиск контактов по имени | Пользователь называет имя, не ID |
| `search_public_chats` | Поиск публичных чатов/каналов | Поиск по username или названию |

### Сообщения

| Инструмент | Что делает | Когда использовать |
|-----------|-----------|-------------------|
| `list_messages` | Сообщения из чата (limit, from_date, to_date) | Основной инструмент чтения |
| `get_messages` | Конкретные сообщения по ID | Нужно прочитать конкретное сообщение |
| `search_messages` | Поиск по тексту в чате | Найти что-то конкретное |
| `send_message` | Отправить сообщение | Только по явной просьбе пользователя |

### Профиль

| Инструмент | Что делает |
|-----------|-----------|
| `get_me` | Профиль текущего пользователя |
| `get_user_status` | Онлайн-статус контакта |

## Паттерны работы

### Найти чат по имени

1. `list_chats` (limit: 30) — получить список
2. Найти нужный по названию
3. Использовать его ID для дальнейших операций

Если не нашёл — `search_contacts` по имени или `search_public_chats` по username.

### Прочитать переписку

1. Найти чат (паттерн выше)
2. `list_messages` с chat_id и нужным limit
3. Если нужен конкретный период — использовать `from_date` / `to_date` в формате YYYY-MM-DD

### Поиск по чатам

1. `list_chats` — получить список чатов
2. Для каждого релевантного чата — `search_messages` с запросом
3. Собрать результаты

## Типы чатов

- **Личные сообщения (DM)** — 1 на 1, самый частый кейс
- **Группы** — несколько участников, есть `get_participants`
- **Каналы** — односторонняя публикация, комментарии в связанной группе
- **Боты** — автоматические ответы

## Ограничения

- **Rate limits**: Telegram ограничивает частоту запросов. Не делай больше 3-5 запросов подряд без паузы. Если получил FloodWait — подожди указанное время.
- **Лимит сообщений**: за раз можно получить до ~100 сообщений. Для большего объёма — пагинация через offset.
- **Медиа**: текст сообщений доступен, но файлы/фото нужно скачивать отдельно через `download_media`.
- **Отправка**: НИКОГДА не отправляй сообщения без явной просьбы пользователя. Чтение — ок, отправка — только по команде.

## Частые ошибки

- **Chat not found**: ID чата может быть отрицательным для групп/каналов. Всегда бери ID из `list_chats`.
- **Пустой результат поиска**: попробуй другие формулировки, поиск работает по точному вхождению.
- **Нет доступа**: пользователь может не состоять в чате. Просто сообщи об этом.
